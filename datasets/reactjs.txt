# ReactJS [BASIC WORKFLOW]: Components, Props, and useState

React enables developers to build user interfaces by combining small, reusable **functional components**. Components accept custom inputs called **props** and manage local mutable data using the **useState** hook.

**Key Core Concepts:**
* **Functional Components:** JavaScript functions that return JSX (HTML-like syntax executed in JavaScript).
* **Props:** Read-only attributes passed down from parent components to customize children.
* **useState Hook:** Returns the current state value and a setter function to schedule a re-render.
* **Unidirectional Flow:** Data always flows down, ensuring state changes are predictable and traceable.

```javascript
import React, { useState } from 'react';

// Child component receiving props
function GreetCard({ username, userRole }) {
  return (
    <div className="card">
      <h2>Welcome, {username}!</h2>
      <p>Role: <strong>{userRole}</strong></p>
    </div>
  );
}

// Parent component managing state
export default function App() {
  const [role, setRole] = useState('Guest');

  return (
    <div className="container">
      <GreetCard username="Developer" userRole={role} />
      <button onClick={() => setRole('Administrator')}>Elevate Role</button>
    </div>
  );
}
```

# ReactJS [EXPERT ARCHITECTURE]: Custom Rendering Micro-States and useSyncExternalStore

In highly complex layouts, rendering bottlenecks occur when global states (e.g. Context API) force deep component trees to re-render. Modern React architecture utilizes **useSyncExternalStore** to subscribe to external stores, enabling targeted micro-state updates without re-rendering the whole page.

**Key Core Concepts:**
* **Micro-States:** Decoupled states that update components in isolation.
* **External Store Subscription:** Subscribing to mutable stores outside React's context tree.
* **useSyncExternalStore:** React hook that synchronizes external state stores safely with rendering logic, preventing UI tearing.
* **Ref-Based Stores:** Bypassing re-renders for write operations, notifying subscribers only when explicit read keys update.

```javascript
import React, { useRef, useSyncExternalStore } from 'react';

// Custom external store implementation
class MicroStore {
  constructor(initialState) {
    this.state = initialState;
    this.listeners = new Set();
  }
  subscribe = (listener) => {
    this.listeners.add(listener);
    return () => this.listeners.delete(listener);
  };
  getSnapshot = () => this.state;
  setState = (nextState) => {
    this.state = typeof nextState === 'function' ? nextState(this.state) : nextState;
    this.listeners.forEach((listener) => listener());
  };
}

const store = new MicroStore({ count: 0 });

export default function Counter() {
  const currentStore = useSyncExternalStore(store.subscribe, store.getSnapshot);

  return (
    <div>
      <h3>Count: {currentStore.count}</h3>
      <button onClick={() => store.setState({ count: currentStore.count + 1 })}>
        Increment Micro-state
      </button>
    </div>
  );
}
```

# ReactJS [OPTIMIZATION & ANTI-PATTERNS]: Preventing Memory Leaks and Unnecessary Renders

React applications running on low-resource machines easily crash or freeze due to memory leaks, uncleaned event listeners, and redundant renders. Correctly optimizing hooks ensures optimal CPU performance.

**Key Core Concepts:**
* **Effect Cleanup:** Always return a cleanup function in `useEffect` to clear listeners, intervals, and subscriptions.
* **useCallback & useMemo:** Memoize callbacks and calculations to prevent reference changes during component re-renders.
* **Dependency Arrays:** Correctly list variables in hook dependencies to avoid infinite loops and stale closures.

```javascript
// ==========================================
// 🛑 BUGGY/UNOPTIMIZED CODE
// ==========================================
import React, { useState, useEffect } from 'react';

export function BuggyTimer() {
  const [seconds, setSeconds] = useState(0);

  // BUG 1: Missing cleanup in setInterval leads to memory leaks and infinite interval spawn
  // BUG 2: Missing dependency array runs effect on every single render
  useEffect(() => {
    setInterval(() => {
      setSeconds(seconds + 1); // Stale closure bug (uses initial value)
    }, 1000);
  });

  return <div>Timer: {seconds}s</div>;
}

// ==========================================
// 🚀 HIGH-PERFORMANCE SOLUTION
// ==========================================
export function OptimizedTimer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    // FIX 1: Use functional update state to avoid dependencies on external stale closures
    // FIX 2: Return a cleanup function to clear interval when component unmounts
    // FIX 3: Empty dependency array ensures effect runs only once on mount
    const interval = setInterval(() => {
      setSeconds((prev) => prev + 1);
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return <div>Timer: {seconds}s</div>;
}
```
