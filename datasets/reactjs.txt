# What is React and the Virtual DOM

React is a declarative, efficient, and flexible JavaScript library for building user interfaces. It allows developers to compose complex UIs from small, isolated pieces of code called **components**.

**Core Concepts:**
* **Declarative Programming:** You describe *what* the UI should look like based on the current state, and React handles updating the DOM to match it.
* **Virtual DOM:** A lightweight representation of the real browser DOM kept in memory.
* **Reconciliation:** The process where React compares the Virtual DOM with the real DOM using a diffing algorithm and updates only what has changed.
* **Reactivity:** State updates trigger components to re-render automatically.

```javascript
// A simple React elements tree created programmatically
import React from 'react';

const element = (
  <div className="app-container">
    <h1>Welcome to React</h1>
    <p>React manages rendering efficiently using the Virtual DOM.</p>
  </div>
);

export default element;
```

# How to create Functional Components and use Props

Functional components are simply JavaScript functions that accept a **props** object as their first argument and return React elements (JSX). They are the modern standard for writing components in React.

**Core Concepts:**
* **Component Reusability:** Code once, instantiate multiple times with different configurations.
* **Props (Properties):** Immutable, read-only data passed from a parent component to a child component.
* **Unidirectional Data Flow:** Data always flows down from parent to child, making the app easier to trace and debug.
* **JSX Syntax:** A syntax extension that allows writing HTML-like tags directly inside JavaScript.

```javascript
import React from 'react';

// Child component accepting props
function UserProfile({ username, role, status }) {
  return (
    <div className="profile-card">
      <h2>Username: {username}</h2>
      <p>Role: <strong>{role}</strong></p>
      <p>Status: {status === 'active' ? '🟢 Online' : '🔴 Offline'}</p>
    </div>
  );
}

// Parent component passing props
export default function App() {
  return (
    <div className="app">
      <UserProfile username="Alice" role="Developer" status="active" />
      <UserProfile username="Bob" role="Designer" status="offline" />
    </div>
  );
}
```

# Working with State and the useState Hook

State represents the local, mutable memory of a component. The **useState** hook is used to add local state variables to functional components, making them interactive and reactive.

**Core Concepts:**
* **Component Memory:** State persists values across component re-renders.
* **State Hook:** `useState` returns an array containing the current state value and a setter function to update it.
* **Triggering Re-renders:** Calling the state setter function schedules a component re-render with the new value.
* **Encapsulation:** Component state is completely private and isolated from other instances.

```javascript
import React, { useState } from 'react';

export default function Counter() {
  // Declare a state variable 'count' initialized to 0
  const [count, setCount] = useState(0);

  return (
    <div className="counter-container">
      <h3>Dynamic Counter</h3>
      <p className="count-display">Current Count: {count}</p>
      <div className="button-group">
        <button onClick={() => setCount(count + 1)}>Increment</button>
        <button onClick={() => setCount(count - 1)}>Decrement</button>
        <button onClick={() => setCount(0)}>Reset</button>
      </div>
    </div>
  );
}
```

# Handling Side Effects with the useEffect Hook

Side effects are operations that reach outside the React rendering flow, such as fetching data from an API, setting up subscriptions, or manually manipulating the DOM. The **useEffect** hook handles these side effects in functional components.

**Core Concepts:**
* **Effect Callback:** The function containing the side-effect code.
* **Dependency Array:** Controls when the effect runs. Empty `[]` runs once on mount; placing values `[val]` runs the effect when those values change.
* **Cleanup Function:** Returned by the effect callback to clean up subscriptions or timers before unmounting or re-running the effect.

```javascript
import React, { useState, useEffect } from 'react';

export default function DataFetcher() {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // API data fetching operation
    fetch('https://jsonplaceholder.typicode.com/users')
      .then(response => response.json())
      .then(users => {
        setData(users);
        setLoading(false);
      })
      .catch(err => console.error("Fetch error:", err));
  }, []); // Run only once when the component mounts

  if (loading) return <p>Loading user list...</p>;

  return (
    <ul className="user-list">
      {data.map(user => (
        <li key={user.id}>{user.name} ({user.email})</li>
      ))}
    </ul>
  );
}
```

# Creating Custom React Hooks

Custom Hooks are reusable JavaScript functions that encapsulate complex stateful logic, allowing you to extract component behavior into shareable modules. They are named starting with "use" and can call other React hooks.

**Core Concepts:**
* **Logic Reusability:** Share stateful logic between different components without changing component hierarchy.
* **Encapsulation:** Each call to a custom hook has a completely isolated state.
* **Naming Convention:** Must start with the prefix `use` (e.g. `useFetch`, `useAuth`, `useWindowSize`).

```javascript
import { useState, useEffect } from 'react';

// Custom Hook to track browser window size
export function useWindowSize() {
  const [windowSize, setWindowSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight,
  });

  useEffect(() => {
    function handleResize() {
      setWindowSize({
        width: window.innerWidth,
        height: window.innerHeight,
      });
    }

    // Bind event listener
    window.addEventListener('resize', handleResize);

    // Return cleanup function to remove event listener
    return () => window.removeEventListener('resize', handleResize);
  }, []); // Run on mount and clean up on unmount

  return windowSize;
}
```
