# What is .NET and the CLR Runtime

.NET is a free, cross-platform, open-source developer platform built by Microsoft for creating many different types of applications. It relies on the **Common Language Runtime (CLR)** to execute code.

**Core Concepts:**
* **Common Language Runtime (CLR):** The execution engine that handles garbage collection, security, JIT compilation, and thread management.
* **Just-In-Time (JIT) Compiler:** Compiles Intermediate Language (IL) byte-code into native machine instructions on the fly.
* **Base Class Library (BCL):** A massive library of reusable classes, interfaces, and value types for standard tasks (I/O, threading, text, database access).
* **Cross-Platform:** Runs natively on Windows, Linux, macOS, iOS, Android, and WebAssembly.

```javascript
// A simple C# Console Application entry point
using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello from .NET Core!");
        }
    }
}
```

# Building APIs with ASP.NET Core Controllers

ASP.NET Core provides a robust model-view-controller (MVC) pattern for building RESTful Web APIs, allowing endpoints to bind securely to C# classes and methods.

**Core Concepts:**
* **ApiController Attribute:** Configures a class to serve HTTP API requests, enabling automatic model validation and HTTP response binding.
* **Routing Engine:** Maps incoming request URLs to controller methods using route templates.
* **HTTP Action Verbs:** Standard HTTP verb bindings (`[HttpGet]`, `[HttpPost]`, `[HttpPut]`, `[HttpDelete]`) mapping endpoints to methods.

```javascript
using Microsoft.AspNetCore.Mvc;
using System.Collections.Generic;

[ApiController]
[Route("api/[controller]")]
public class UsersController : ControllerBase
{
    // GET api/users
    [HttpGet]
    public ActionResult<IEnumerable<string>> GetUsers()
    {
        return Ok(new string[] { "Alice", "Bob", "Charlie" });
    }
}
```

# Database Access with Entity Framework Core

Entity Framework Core (EF Core) is a lightweight, cross-platform Object-Relational Mapper (ORM) that enables .NET developers to interact with relational databases using C# objects.

**Core Concepts:**
* **DbContext:** The primary class that coordinates EF Core functionality, representing a session with the database.
* **DbSet:** Represents an in-memory collection of entity objects that maps directly to a database table.
* **LINQ (Language Integrated Query):** Syntactic queries integrated into C# used to retrieve and filter data from the DB Context.
* **Migrations:** Auto-generated schema changes tracking entity changes and applying incremental SQL script updates.

```javascript
using Microsoft.EntityFrameworkCore;
using System.Linq;

// Database Context Setup
public class AppDbContext : DbContext
{
    public DbSet<User> Users { get; set; }
}

public class User
{
    public int Id { get; set; }
    public string Name { get; set; }
}

// Querying database using LINQ
public class UserService
{
    private readonly AppDbContext _context;

    public UserService(AppDbContext context)
    {
        _context = context;
    }

    public User GetActiveUser(int id)
    {
        return _context.Users.FirstOrDefault(u => u.Id == id);
    }
}
```

# Dependency Injection in .NET Core

Dependency Injection (DI) is built directly into .NET Core. It provides a native container to manage object lifecycles, reducing class coupling and increasing testability.

**Core Concepts:**
* **Service Registration:** Registering services in the application startup configuration.
* **Lifecycles:** 
  * *Transient:* Created every time they are requested.
  * *Scoped:* Created once per client request (connection).
  * *Singleton:* Created once on startup and shared globally.
* **Constructor Injection:** Automatically resolving and injecting dependencies into controller or service constructors.

```javascript
using Microsoft.Extensions.DependencyInjection;

// Startup configuration (Program.cs)
var builder = WebApplication.CreateBuilder(args);

// Registering dependencies with Scoped and Singleton lifecycles
builder.Services.AddScoped<IUserRepository, UserRepository>();
builder.Services.AddSingleton<IEmailService, EmailService>();

var app = builder.Build();
```

# Asynchronous Programming with Async/Await

Asynchronous programming in C# utilizes the `async` and `await` keywords to prevent blocking application threads during heavy I/O operations (like database or network calls).

**Core Concepts:**
* **Task-based Asynchronous Pattern (TAP):** Methods returning a `Task` or `Task<T>` representing operations that will complete in the future.
* **Await Keyword:** Suspends thread execution without blocking the CPU, yielding control back to the runtime thread pool until the task completes.
* **Non-blocking Threads:** Prevents UI freezes in desktop apps and increases thread throughput on high-traffic web servers.

```javascript
using System.Net.Http;
using System.Threading.Tasks;

public class ApiClient
{
    private static readonly HttpClient _client = new HttpClient();

    // Asynchronously fetches data from a remote endpoint
    public async Task<string> FetchJsonDataAsync(string url)
    {
        HttpResponseMessage response = await _client.GetAsync(url);
        response.EnsureSuccessStatusCode();
        string data = await response.Content.ReadAsStringAsync();
        return data;
    }
}
```
