# What is Express and Node.js

Express is a minimal and flexible web application framework for Node.js. It runs on top of the native Node.js HTTP module, simplifying routing, request processing, and endpoint mapping.

**Core Concepts:**
* **Event-Driven Runtime:** Node.js executes JavaScript on the server using Google's V8 engine, utilizing a single-threaded event loop.
* **Non-Blocking I/O:** Heavy operations (files, network requests) run asynchronously, preventing the server thread from hanging.
* **Unopinionated Framework:** Express provides core utility features without enforcing database models, folder structure, or third-party packages.

```javascript
// Simple Express Web Server Setup
import express from 'express';
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.send('Welcome to Node.js & Express!');
});

app.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});
```

# Middleware Architecture and Execution Chain

Middleware functions are the fundamental building blocks of an Express application. They process requests sequentially before they reach the final route handler.

**Core Concepts:**
* **Middleware Signature:** Functions accepting three parameters: `req` (request object), `res` (response object), and `next` (next middleware callback).
* **Next Callback:** Invoking `next()` tells Express to pass control to the subsequent middleware function in the execution queue.
* **Request Modification:** Middleware can run logic, read headers, modify request variables, or terminate the pipeline early by sending responses.

```javascript
import express from 'express';
const app = express();

// Custom Logger Middleware
function requestLogger(req, res, next) {
  const method = req.method;
  const path = req.path;
  const timestamp = new Date().toISOString();
  console.log(`[${timestamp}] ${method} request to ${path}`);
  next(); // Pass control to next handler
}

app.use(requestLogger); // Apply globally to all routes
```

# Routing and Dynamic URL Parameters

Routing maps URL paths and HTTP verbs (GET, POST, PUT, DELETE) to specific handlers. Dynamic URL parameters allow you to extract variables directly from matching paths.

**Core Concepts:**
* **HTTP Verbs:** Routes are defined mapping to standard HTTP verbs (`app.get`, `app.post`, `app.put`, `app.delete`).
* **Route Params:** Parameters marked with a colon (e.g. `/users/:id`) are parsed dynamically and accessed via `req.params`.
* **Query Params:** URL query parameters (e.g. `?sort=desc`) are accessed via the `req.query` object.

```javascript
import express from 'express';
const app = express();

// Route with dynamic ID parameter and query filter
app.get('/api/users/:id', (req, res) => {
  const userId = req.params.id;
  const sortBy = req.query.sort || 'asc';
  res.json({
    userId,
    sortBy,
    message: `Details fetched for User #${userId}`
  });
});
```

# Building REST APIs and JSON payload parsing

Express is commonly used to build RESTful APIs that exchange data using JSON payloads. Express requires built-in parser middleware to parse incoming request payloads.

**Core Concepts:**
* **Body Parser:** Native `express.json()` middleware parses incoming JSON requests and populates the `req.body` object.
* **Response Status Codes:** Returning correct HTTP status codes (200 for OK, 201 for Created, 400 for Bad Request).
* **JSON Serialization:** Using `res.json()` automatically serializes objects into JSON strings and sets correct content-type headers.

```javascript
import express from 'express';
const app = express();

app.use(express.json()); // Enable JSON body parsing

app.post('/api/products', (req, res) => {
  const { name, price } = req.body;
  if (!name || !price) {
    return res.status(400).json({ error: 'Name and Price are required.' });
  }
  res.status(201).json({
    message: 'Product created successfully',
    product: { id: Date.now(), name, price }
  });
});
```

# Handling CORS (Cross-Origin Resource Sharing)

CORS is a security mechanism enforced by web browsers to restrict web apps from requesting resources from different origins. Express configures CORS using HTTP headers.

**Core Concepts:**
* **Origin Headers:** The browser sends origin headers to the server, and the server decides whether to authorize the origin.
* **Allow Headers:** Server configuration explicitly permitting source domains, authorized HTTP methods, and custom headers.
* **Pre-flight Requests:** Browsers send OPTIONS requests to check CORS configurations before executing write operations (POST, PUT, DELETE).

```javascript
import express from 'express';
const app = express();

// Custom CORS middleware configuration
app.use((req, res, next) => {
  res.setHeader('Access-Control-Allow-Origin', 'http://localhost:3000'); // Authorize frontend origin
  res.setHeader('Access-Control-Allow-Methods', 'GET, POST, OPTIONS, PUT, DELETE');
  res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
  
  if (req.method === 'OPTIONS') {
    return res.sendStatus(200); // Handle pre-flight checks
  }
  next();
});
```
